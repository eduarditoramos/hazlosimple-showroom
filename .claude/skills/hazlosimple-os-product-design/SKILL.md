# HazloSimple OS Product Design Skill

Use this skill when working on HazloSimple OS UI, Home v2, microdemos, dashboard composition, or product design.

## Mission
Help build HazloSimple OS as a premium product showroom / command center for mini business apps.

The goal is not decoration. The goal is commercial clarity.

A user should understand in 3 seconds:
- This is not a normal landing page.
- These are mini apps for real businesses.
- Each app solves a specific operational pain.
- They can request a custom app for their business.

## Core Product Principle
Do not show data. Show a business functioning.

Each screen must communicate:
- metric
- flow
- chart
- active record
- next action

## Aesthetic Guardrails
Do:
- use strong hierarchy
- use clean sans typography
- use mono only for small system labels
- use navy/green/off-white/charcoal
- use app accents intentionally
- use inline SVG or sober Unicode symbols
- use meaningful charts and visual flows
- design like a senior product designer building a SaaS demo

Do not:
- make generic SaaS cards
- make fake Mac cosplay
- use orange as a dominant color
- overuse monospace
- use emojis as icons
- add random decorations
- add empty panels
- add backend/auth/payments
- install templates that override identity

## App Mental Models
CRM = mini Salesforce:
conversion, pipeline, follow-up urgency, probability, value.

Cocina / POS = mini Toast/Square/KDS:
orders by state, active comanda, prep time, sales, delayed order.

Clínica / Pacientes = mini Doctoralia/consultorio:
agenda, patient file, instructions, confirmation, follow-up.

Control Operativo = mini Odoo/ERP:
orders, stock, responsible person, task queue, operational blockers.

## Home v2 Command Center
When redesigning Home, prefer:
- one strong command center composition
- fewer borders
- less table-like UI
- more grouped product modules
- charts that communicate movement
- clear primary CTA
- app dock/launcher as navigation
- live preview that feels operational

## Acceptance Criteria
A good result:
- feels like product, not template
- makes each app recognizable in 3 seconds
- shows why someone would pay
- has no large useless empty space
- guides the eye clearly
- builds desire to click "Quiero mi app"

A bad result:
- looks like admin table
- looks like generic dashboard
- depends only on labels
- has too many borders
- has weak typography
- has random colors
- has static cards pretending to be apps
